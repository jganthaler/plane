Getting started

Overview

Plane is a starter web project designed to provide a solid foundation for developing any type of web project. It includes a Gulp-based build system for automating tasks such as compiling SCSS to CSS, concatenating, transpiling, and minifying JavaScript, along with other essential features such as a flexbox grid system and basic settings.

Prerequisites

Node.js and npm installed on your machine.

Installation

  1. Clone the repository: git clone [repository-url]
  2. Navigate to the project directory: cd [project-name]
  3. Install dependencies: npm install

Usage

  1. Run npm start for development mode or npm run build for production mode.
  2. Edit the src/scss/settings/_settings.scss file for basic configuration or edit any other file as you wish.

Project structure

The Plane framework is organized to keep your styles and scripts modular and easy to maintain. Below is the structure of the project:

/project-name
│   README.md              # Project documentation
│   gulpfile.js            # Gulp tasks for building and watching files
│   package.json           # Node dependencies and scripts
│   index.html             # Entry point for the project
└───src
    └───scss               # Source files for styles
    │   └───base           # Global styles and helpers (reset, typography, helpers, grid, etc.)
    │   └───components     # Reusable UI components (menus, breadcrumbs, cards, etc.)
    │   └───elements       # Styles for HTML elements (buttons, tables, etc.)
    │   └───layout         # Layout styles (sidebar, navbar, footer etc.)
    │   └───pages          # Page-specific styles (login, signup, etc.)
    │   └───settings       # Global variables and settings
    │   └───util           # SCSS tools and helpers (functions, mixins etc.)
    │   └───vendor         # Third-party or external styles (optional)
    │   │   index.scss     # Main SCSS file that imports all other SCSS files
    └───js                 # Source files for JavaScript
        │   main.js        # Main JavaScript file
        │   ... other JS files
└───dist
    └───css                # Compiled and minified CSS
    └───js                 # Compiled and minified JavaScript

Settings

The settings folder is the core configuration folder for the Plane framework. It defines key global variables such as colors, typography, grid settings, and breakpoints that are used throughout the project. The rule of thumb for this folder is that it should not output a single line of CSS when compiled on its own. These are nothing but SCSS variables.

The main sections if the _settings.scss file include:

  • Global Settings: Default settings for font sizes, colors, line heights, etc.
  • Breakpoints: Configurations for screen sizes for responsive design.
  • Grid: Configurations for the grid system (number of columns, gutters, etc.)
  • Typography: Default settings for paragraphs, headings, and other text elements.
  • Form: Configurations for form elements such as inputs, labels, and buttons.

This file is designed to be modified according to your project’s needs, allowing for easy customization of the core design system.

Base

The base folder contains foundational styles that set the groundwork for the Plane framework. These styles ensure consistency across the entire project and include the following:

  • Reset CSS: Resets default browser styling to create a consistent starting point.
  • Global Styling: Basic global styles for elements like body, headings, and links.
  • Helper Classes: Utility classes for margins, padding, alignment, and other layout helpers.
  • Typography: Base typography settings including font sizes, weights, and line heights for headings and paragraphs.
  • Form Styles: Standard form element styling such as inputs, labels, and buttons.
  • Grid System: Flexbox-based grid system for responsive layouts.

This folder is essential for setting up the core design language and basic structure of your project.

Grid

The Plane framework offers a flexible and responsive 12-column grid system. This system is built using Flexbox and allows for easy customization of layouts that adapt to different screen sizes.

Basics

The Plane framework’s grid system utilizes two main classes: .grid-x and .cell.

  • .grid-x: Creates a flex container for aligning grid cells in a row.
  • .cell: Defines individual grid items and their column spans.

To specify the number of columns a cell should span at different breakpoints, add classes in the format {name-of-breakpoint}-{number-of-columns}. This allows for responsive adjustments based on screen size.

<div class="grid-x">
    <div class="cell medium-6 large-4">
        .medium-6 .large-4
    </div>
    <div class="cell medium-6 large-8">
        .medium-6 .large-8
    </div>
</div>
  • medium-6: On medium screens, each cell spans 6 out of 12 columns (50% width).
  • large-4: On large screens, the first cell spans 4 out of 12 columns (33.33% width).
  • large-8: On large screens, the second cell spans 8 out of 12 columns (66.67% width).

Gutters

To add gutters between cells in the grid, use the grid-margin-x class on your grid. The size of the gutters can be customized in the _settings.scss file, allowing you to adjust the spacing to fit your design needs.

Grid Containers

The grid is designed to take up the full width of the available space by default. To restrict it horizontally, use the grid-container class. This will center the grid and set a maximum width, based on the $grid-container variable (default is 1200px), with left and right padding determined by half of the $grid-container-padding setting.

  • Fluid Container: To make the grid stretch across the full width of the available space, add the fluid class to your grid-container.
  • Full Container: To stretch the grid across the full width and remove the container padding, use the full class with your grid-container.

Offsets

To use offsets, add a class like large-offset-4 to your cell. This class will add a left margin to the cell, pushing it to the right by a specified number of columns.

Typography

Typography sets default styles for headings, paragraphs, semantic text, blockquote and lists elements.

  • By default there are different font sizes for small and larger screens. You can set it for the body in _base.scss file.
  • Set default text styles in the _text.scss file. Different leadings and lineheights depending on the screen size can also be configured here.
  • Set the global link color via $anchor-color in the _settings.scss file.

Helper

Plane provides a variety of helper classes for common layout and text styling needs, including flex alignment, text alignment, text transformation, and margin/padding utilities.

Flex Alignment

Use these classes to align flexbox items within a container:

  • .align-left: Aligns items to the start (left).
  • .align-right: Aligns items to the end (right).
  • .align-center: Centers items horizontally.
  • .align-justify: Distributes items evenly, with space between.
  • .align-spaced: Distributes items evenly, with space around.

Text Alignment

Use these classes to align or transform text:

  • .text-left: Aligns text to the left.
  • .text-center: Centers text.
  • .text-right: Aligns text to the right.
  • .text-uppercase: Transforms text to uppercase.
  • .text-lowercase: Transforms text to lowercase.
  • .strong: Makes text bold.
  • .italic: Makes text italic.

Margin and Padding

Adjust margins and paddings responsively using these utility classes. They are generated based on your project's global spacing settings:

  • .margin-top, .margin-bottom: Adjusts top/bottom margins.
  • .padding-top, .padding-bottom: Adjusts top/bottom padding.
  • Variations available for small, medium, and large spacings (e.g., .margin-large-top, .padding-small-bottom).

Components

Components are reusable UI elements like menus, breadcrumbs, cards, and more. With Plane, you have the flexibility to modify, extend, and create new components as needed, allowing you to tailor the framework to suit your requirements.

Menu

The menu component can be displayed either horizontally or vertically. Add the class .is-vertical to the menu to create a vertical menu. 

<ul class="menu">
    <li><a href="#">One</a></li>
    <li class="is-active"><a href="#">Two</a></li>
    <li><a href="#">Three</a></li>
    <li><a href="#">Four</a></li>
</ul>

  • Two
  • Three

Breadcrumb

The breadcrumb component is used for navigation, indicating the user's current location within the website hierarchy.

<ol class="breadcrumb">
    <li><a href="#">Home</a></li>
    <li><a href="#">Features</a></li>
    <li class="is-disabled"><span>Disabled</span></li>
    <li class="is-active"><a href="#">Active</a></li>
</ol>
  1. Home
  2. Features
  3. Disabled
  4. Active

Utilities

The util folder of the project contains a set of SCSS helper functions and mixins aimed at enhancing the development process by providing commonly used functionalities.

SCSS Helper Functions

  • rem-calc($values, $base: null): Converts one or more pixel values into matching rem values, optionally based on a given base font size.
  • strip-unit($number): Returns the number without unit

SCSS Mixins

  • breakpoint($val: $zero-breakpoint): Generates a media query string based on the input value, which can be a breakpoint name, pixel value, or keyword.

  • vertical-center():
